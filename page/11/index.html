.gitignore 基础知识

发表于 2018-04-13 分类于 Git 阅读次数： 本文字数： 855 阅读时长 ≈ 1 分钟

.gitignore顾名思义是Git中用来管理所需要忽略或者说不用纳入版本控制文件

基本配置语法

  1. “#“：表示注释
  2. “/“：表示目录
  3. “*“：表示通配符，用来通配多个字符
  4. “?“：表示通配单个字符
  5. “[]“：表示包含单个字符的匹配列表
  6. “!“：表示不忽略匹配到的文件或者目录
